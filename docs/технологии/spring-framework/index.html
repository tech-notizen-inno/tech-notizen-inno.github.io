    Контейнер Spring IOC / Dependency Injection#

    Схема работы приложения Spring

    • getBean, getResource, getEnvironment - получение внедренных сущностей
    • Метод, который возвращает бин аннотируется @Bean для Spring.
    • @Configuration is the annotation of Configuratin annotated on top of a class + @Bean annotation that’s annotated on top of a method signaling to the Spring Framework that this method contains a definition of Java Bean
    • Объект аннотируется @Component → Spring создаст его
    • Объект имеет аргумент конструктора @Autowired → Spring автоматически внедрит источник данных, настроенный с помощью вашего метода @Bean (В новых версиях достаточно конструктора с параметром, который является Bean)
    • Если в ваших конфигурациях Spring не было настроено ни одного источника данных, вы получите исключение NoSuchBeanDefinition во время выполнения.
    • Объявление конфигурации контекста с использованием области аннотирования
      • Scope(“singleton”) → Ваш бин будет синглтоном, т.е. будет только один экземпляр.
      • Scope(“prototype”) → Каждый раз, когда кому-то нужна ссылка на ваш компонент, Spring создает новый.
      • Scope(“session”) → Для каждого сеанса HTTP пользователя будет создан один компонент.
    • @EnableAutoConfiguration - конфигурация на основании добавленных jar-зависимостях
    • @Autowired, @Qualifier, @Component -dependancy ingection

    Spring MVC#

    Uber-контроллер в среде Spring MVC Spring представляет собой сервлет, который называется DispatcherServlet. Объявление контроллера. Все что проаннотировано @Controller сообщает Spring: этот класс хочет реагировать на HTTP-запросы и ответы

    • @RequestMapping передает информацию о “маршрутизации”. Маппинг метода на пусть
    • @RestController - класс типа контроллер, который работает с content negotiation
    • @GetMapping / @PostMapping - Annotation for mapping HTTP GET and POST requests onto specific handler methods.  - @PathParam, @PathValue - данные выбираются из URL запроса HTTP  - Аннотацию @ResponseBody в методе контроллера сообщает Spring, что вы хотите записать свой Java-объект непосредственно в HttpResponse (например, в виде XML или JSON).  - Content Negotiation  - Указав заголовок Accept, такой как «Accept: application/json» или «Accept: application/xml». Это работает “из коробки”, но требует определенных библиотек в вашем пути к классам для поддержки сортировки XML или JSON.  - Добавив расширение URL к вашему пути запроса, например /health.json или /health.xml. Это требует настройки на стороне Spring MVC для работы.  - Добавив параметр запроса в путь запроса, например /health?Format = json. Это требует настройки на стороне Spring MVC для работы.   

    Spring Boot#

    Spring Boot — это полезный проект, целью которого является упрощение создания приложений на основе Spring. Он позволяет наиболее простым способом создать web-приложение, требуя от разработчиков минимум усилий по его настройке и написанию кода

    Управление зависимостями#

    Spring Bootупаковывает необходимые сторонние зависимости для каждого типа приложения на основе Spring и предоставляет starter-пакетов (spring-boot-starter-web, spring-boot-starter-data-jpa и т.д.)

    •  spring-boot-starter-data-jpa - БД
    •  spring-boot-starter-web - web-приложение
    • spring-webmvc, jackson-json, validation-api - Spring MVC

    Документация по Spring Boot starter-пакетами

    Spring Data#

    • @Repository - шаблон DAO  - @Entity - основаная аннотация JPA

      • Entity класс должен содержать public или protected конструктор без аргументов (он также может иметь конструкторы с аргументами),

      • Entity класс должен быть классом верхнего уровня (top-level class)

      • Entity класс не может быть enum или интерфейсом,

      • Entity класс не может быть финальным классом (final class) @Entity @Table(name="user") class User { @Id @GeneratedValue public Integer id;

             `@Column(name="name", unique=true, length=100)`
     `public String name;`

     `@Column(name="level", nullable=true)`
     `public Integer level;`
  `}`

    Spring Actuator#

    • подключить через Gradle dependencies { implementation("org.springframework.boot:spring-boot-starter-actuator") }
    • подключение к мониторингу через HTTP
      • http://localhost:8080/actuator - все endpoints
      • /actuator/{id} доступ к конкретному endpoint
    • Список endpoints для мониторинга
    • конфигурирование
      • management.endpoint.{id}.enabled - включить/выключить любую точку. Пример включения shutdown - management.endpoint.shutdown.enabled=true
      • management.endpoints.enabled-by-default=false/true - включение/отключение по умолчанию endpoints через JMX
      • management.endpoints.web.exposure.include - включение endpoints. Пример - management.endpoints.web.exposure.include= health,info,env.
        • Включит все endpoints - management.endpoints.web.exposure.include=*
      • management.server.port=-1 - отключить все endpoints
      • Настроить доступ по другому URL - management.endpoints.web.base-path. Пример - management.endpoints.web.base-path=/monitor
      • Отключить преднастроенные индикаторы здоровья - management.health.defaults.enabled=false
    • преднастроенные индикаторы здоровья (HealthIndicators)
      • DiskSpaceHealthIndicator diskspace
      • MongoHealthIndicator mongo
      • CassandraHealthIndicator cassandra
      • DataSourceHealthIndicator datasource
    • Код для создания custom health indicator
