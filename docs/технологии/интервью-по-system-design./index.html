    Общие подходы#

    • Прохождение интервью
      • Внешний контур системы
        • Actors
        • Внешние интеграции
        • Основные блоки внутри системы
      • Ограничения по производительности 
        • SLA
        • query per second
        • storage
      • Определить основной поток работы (краткое описание)
      • Детализация по кубикам (Architecture Building Blocks)
        • API-Gateway/Load Balancing 
        • Services
        • DB 
          • Cluser (Write/Read)
          • Выбрать тип БД (SQL / NoSQL)
          • Sharding / Constistance
          • Есть есть транзакционность - какой уровень транзакции (Repeataable Read), уровень блокировки (Оптимистичные и Пессимистичные блокировки)
          • Cashing
      • Доменная модель (для хранение в БД) - как данные будет использованы на разных стадиях
      • На клиенте возможно использовать uid , чтобы не было повторного запроса

    Оценка#

    1. Как кандидат формализовал задачу (разобрался в требованиях)
    2. Как очертил границы
    3. Масштабирование под нагрузку
    4. Схема должна быть читаема без автора
    5. Двигаться методично
    6. Задавать вопросы - не брать на себя больше работы, чем требуется
    7. Кандидат также может задать вопросы
    8. Sequence диаграммы обычно рассматриваются после общего построения>

    System Design Framework#

    Четыре стадии интервью Описать систему в целом Контракты в стиле REST Определить требования к производительности, SLA Используя общее понимание производительности Обсудить более сложные моменты (выбор типа БД, масштабирование, abnormal cases) Сложные моменты (tricky questions)
