    Термины#

    • LDAP, или Lightweight Directory Access Protocol, является открытым протоколом, используемым для хранения и получения данных из каталога с иерархической структурой.
    • Служба каталогов используется для хранения, организации и представления данных в формате «ключ-значение».
    • Все записи добавляются в систему LDAP в виде веток на деревьях, называемых Data Information Trees, или DIT-ы
    • Сама информация в LDAP-системе в основном хранится в элементах, называемых атрибутами.
    • Атрибуты объединяются в записи (entry).
    • Атрибуты собираются в сущностях, называемых ObjectClass (классы объектов)
      • Два основных типа ObjectClasses — это структурный (structural) и дополнительный (auxiliary).
      • Запись должна должна иметь ровно один структурный класс, но может иметь ноль или более вспомогательных классов, используемых для дополнения списка атрибутов, доступных этому классу.
    • Определения ObjectClass и определения атрибутов сгруппированы в schema (схема)
    • Ссылки на записи делаются по однозоначным атрибутам или группе атрибутов, называемых  RDN (от relative distinguished name).
    • distinguished name (“dn”, “отличительное имя”) записи используется для идентификации записи.
    • Варианты протокола
      • ldap://: Это основной протокол LDAP, позволяющий получить структурированный доступ к службе каталогов.
      • ldaps://: Этот вариант используется для доступа к LDAP поверх SSL/TLS. Обычно трафик LDAP не шифруется. Такой способ шифрования LDAP-соединений устарел, и вместо него рекомендуется использовать шифрование STARTTLS.
      • ldapi://: Это используется для указания LDAP через IPC. Это часто используется для безопасного соединения с локальной LDAP-системой в административных целях. Он связывается через внутренние сокеты вместо того, чтобы использовать открытый сетевой порт.
    • Применяются 2-а типа типа авторизации: Basic, SASL
    • Прослушиваемые порты TCP and UDP port 389
    • Binary protocol

    Операции#

    • LDAP commands include searching, adding, modifying or deleting entries in the database.
    • Операция Bind (to allow authentication information to be exchanged between the client and server). The Bind operation should be thought of as the “authenticate” operation.
    ASN.1
    Bind Request
    Bind Responce
    • В фильтрации допускаются булевые операторы
      • & (...K1...) (...K2...) - логическое И
      • (| (...K1...) (...K2...)) - логического ИЛИ
      • (|(& (...K1...) (...K2...))(& (...K3...) (...K4...))) = (K1 AND K2) OR (K3 AND K4) - вложенные операторы
      • (! (...K1...)) - логическое НЕТ

    Команды#

    • sudo apt install slapd ldap-utils - установить сервер
    • sudo dpkg-reconfigure slapd - запустить конфигурацию
    • sudo systemctl status slapd - проверить статус сервера
    • grep -e 'inetOrg' /etc/ldap/schema/* - поискать вхождение в файлах схемы
    • ldapsearch -x -LLL -b 'ou=Groups,dc=iskripov,dc=local' - поиск в каталоге, подключение локальное
    • ldapsearch -H ldap://localhost -LLL -b 'ou=Users,dc=iskripov,dc=local' -D 'cn=admin,dc=iskripov,dc=local' -w secure - поиск по каталогу
      • -H ldap://localhost - подключение к хосу
      • -D - указание на администратора
      • -w <password> или -W - указание пароля либо запрос пароля интерактивно
    • ldapsearch -H ldap://localhost -LLL -b 'ou=Users,dc=iskripov,dc=local' -D 'cn=admin,dc=iskripov,dc=local' -W '(sn=Skripov)' - поиск с использованием фильтра
      • '(sn=Skripov)' - фильтр
    • ldapadd -H ldap://localhost -D 'cn=admin,dc=iskripov,dc=local' -W -f ./files/add.ldif - выполнить LDIF файл
      • -f ./files/add.ldif - файл LDIF
