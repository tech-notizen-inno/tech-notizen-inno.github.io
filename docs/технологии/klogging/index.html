    • Логирование для Kotlin документация
    • Конфигурирование возможно через ENVIRONMENT VARIABABLES (ссылка), DSL (ссылка), JSON Config (ссылка)
    • Конфигурирование через файл с помощью файлов klogging.conf или klogging.json. Пример файла JSON конфигурации
    • Можно передавать в логгер контексты
      • от базоваого (для всего приложения)
      • до контекста при вызове
    • Два типа логгеров - Klogger (для работы с suspend функциями), NoCoLogger (для работы в обычном контексте).
      • Можно наследоваться от них, чтобы получить доступ к функциям логирования (logger.error, logger.warn, logger.info, logger.debug, logger.trace, logger.log)
      • Можно создавать статический член coobject
      • Создавать в месте вызова

    DSL. Основные методы#

    loggingConfiguration {  
	sink("stdout", RENDER_SIMPLE, STDOUT)  
	sink("seq", seq("http://localhost:5341"))  
	logging {  
		fromLoggerBase("com.example")  
		fromMinLevel(Level.INFO) {  
			toSink("stdout")  
		}  
	}  
	kloggingMinLevel(DEBUG)  
	minDirectLogLevel(INFO)
}
    Оператор Описание
    loggingConfiguration This function creates a configuration for the running program. Опция append=true - расширяет конфигурация из файла
    sink This function configures a named sink with a render and a sender
    logging his function configures logging from specified loggers at specified levels to specified sinks
    fromLoggerBase, exactLogger and matchLogger These functions specify how to match logger names
    fromMinlevel, toMaxLevel, atLevel and inLevelRange These functions specify the levels at which to dispatch log events.
    toSink This function specifies the name of a sink to dispatch events to
    minDirectLogLevel This function specifies the minimum level at which log events are sent direct to sinks instead of being sent asynchronously via coroutine channels
    kloggingMinLogLevel This function sets the minimum level used by the internal logger to decide whether to emit log messages.

    Встренные конфигурации (ссылки)#

    Рендерер       Описание                                    
    RENDER_SIMPLE Отображает лог-события в виде строк с временной меткой в локальной зоне сервера и параметрами в фигурных скобках.
    RENDER_ISO8601 Форматирует лог-события в строковые сообщения с временной меткой в формате ISO8601 (UTC).          
    RENDER_ANSI     Аналог RENDER_SIMPLE, но с использованием выделений цветом для уровней логирования и форматированных колонок.
    RENDER_CLEF     Переводит лог-события в JSON, используя Compact Log Event Format, для отправки на сервер Seq.    
    RENDER_GELF     Преобразует лог-события в JSON в формате Graylog Extended Log Format для отправки на сервер Graylog.
    RENDER_STANDARD Создает стандартный JSON-формат логов с такими полями, как временная метка, уровень, логгер и сообщение.
