    Общие данные#

    • Состоит из 2-х частей
      • osqueryd is the host monitoring daemon that allows you to schedule queries and record OS state changes. The daemon aggregates query results over time and generates logs, which indicate state change according to each query. The daemon also uses OS eventing APIs to record monitored file and directory changes, hardware events, network events, and more.
        • The primary daemon feature is executing a query schedule. This schedule is defined in an osquery configuration
      • osqueryi lets you run meta-commands and query osquery tables.
        • The shell accepts a single positional argument and one of the several output modes.
        • You may also pipe a query as stdin.

    Примеры запросов#

    • SELECT vendor, model FROM usb_devices; - получение USB устройств
    • SELECT DISTINCT process.name, listening.port, process.pid FROM processes AS process JOIN listening_ports AS listening ON process.pid = listening.pid WHERE listening.address = '0.0.0.0'; - получение сокетов в привязке к процессам
    • SELECT * FROM routes WHERE destination = '::1' - получить рауты
    • SELECT pid, name, path FROM processes WHERE pid > 0 LIMIT 1 - получить процессы
    • SELECT * FROM osquery_info; - look at a “meta” table that provides details to osquery about itself. These tables are prefixed with osquery_
    • SELECT p.pid, name, p.path AS process_path, pf.path AS open_path FROM osquery_info i JOIN processes p ON p.pid = i.pid JOIN process_open_files pf ON pf.pid = p.pid WHERE pf.path LIKE '/dev/%' - открытые файлы процессом osquery
    • SELECT * FROM file;, SELECT * FROM file WHERE path = '/dev/zero' - работа с файлами
    • SELECT path, mtime, sha256 FROM file JOIN hash USING (path) WHERE file.directory = '/etc' ORDER BY mtime DESC LIMIT 1; - получить хэш для файла

    Дополнительные команды#

    • to list all tables: .tables
    • to list the schema (columns, types) of a specific table: .schema table_name or pragma table_info(table_name); for more details
    • to list all available commands: .help
    • to exit the console: .exit or ^D
    • select all from a table: .all [TABLE]
    • stop after hitting an error: .bail ON|OFF
    • connect to an osquery extension socket: .connect PATH
    • disconnect from a connected extension socket: .disconnect
    • turn command echo on or off: .echo ON|OFF
    • построчная выдача результата: .mode line

    Передача данных на сервер#

    Настройка osquery для безопасной потоковой доставки логов и конфигурации по TLS (TLS logger + TLS config).

    • Базовая архитектура
      • TLS Config: osquery периодически тянет конфигурацию из endpoint’а.
      • TLS Logger: osquery отправляет результаты scheduled queries и события на endpoint.
    • Основные эндпоинты (пути можно менять):
      • /api/v1/osquery/enroll — регистрация (возврат node_key) Вход: {"enroll_secret":"...", "host_identifier":"..."}.
        Ответ: {"node_key":"<random_token>"}.
      • /api/v1/osquery/config — выдача конфигурации агенту Вход: {"node_key":"..."}.
        Ответ: {"options":{...}, "schedule":{...}, "packs":{...}}.
      • /api/v1/osquery/log — прием логов (results + status) Вход: {"node_key":"...", "log_type":"result","data":"<jsonl>"}
        и/или для status-логов: log_type="status".
        Ответ: {"success": true}.
      • /api/v1/osquery/certs — (опционально) выдача корневого сертификата
    • Конфигурация osqueryd

      • файл флагов /etc/osquery/osquery.flags

      • базовый osquery.conf (может забираться из endpoint (в таком случае не нужен))

      • Запуск sudo osqueryd --flagfile=/etc/osquery/osquery.flags --config_path=/etc/osquery/osquery.conf --verbose --tls_dump=true (2-а последних аргумента для отладки)

      • Пример сервера приема данных

        • дополнительные endpoints для заданий (POST /distributed/read и /distributed/write (опционально): Read: выдает задания; Write: принимает результаты.)
