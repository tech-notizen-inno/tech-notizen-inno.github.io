    Redis Sentinel#

    Redis Sentinel - это сервис, обеспечивающий создание распределённых систем. В основе Sentinel лежит кластер Sentinel-процессов, работающих совместно. Они координируют состояние системы, реализуя конфигурацию высокой доступности Redis. Функции Sentinel

    1. Мониторинг.
    2. ﻿﻿﻿Отправка уведомлений администраторам.
    3. ﻿﻿﻿Управление восстановлением системы после отказа. Узлы Sentinel могут запустить процесс восстановления системы после сбоя в том случае, если ведущий экземпляр Redis недоступен.
    4. ﻿﻿﻿Управление конфигурацией. Узлы Sentinel, кроме того, играют роль системы, позволяющей обнаруживать текущий ведущий экземпляр
      Redis.

    Redis Cluster#

    • Redis использует разреженные шарды, называемые хэш-слотами. Всего около 16К таких слотов. При падении/добавлении ноды переносятся только те записи, которые попали в новый слот
    • Протокол Gossip используется для определения общего состояния кластера. Все эти узлы постоянно обмениваются информацией для того чтобы знать о том, какие шарды доступны и готовы обрабатывать запросы. Кворум позволяет промоутить Slave до Master

    Персистентное хранение#

    Для гарантированной записи на диск используется

    • системный вызов fsync()
    • Создание forks процессов Redis. Процесс-родитель и процесс-потомок используют память совместно (для потомка используется ссылка на память родителя). Для процесса-родителя используется механизм CopyOnWrite, что позволяет копировать только изменившиеся страницы.
