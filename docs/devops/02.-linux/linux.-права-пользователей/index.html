    Linux. Права Пользователей

    • SUID, SGID, Stikybit

    • chown (chown -R frontend:frontend /run/frontend/) - смена прав на файл или директорию

    SUID, SGID, Stikybit#

    • SUID позволяет запустить процесс непривилегированному пользователю с правами владельца файла.
      • EUID — это эффективный идентификатор пользователя, он изменяется для процессов (не для пользователя) с битом setuid, которые выполняет пользователь.
      • chmod u+s,a+rx <command> - назначить права с SUID
      • при создании/записи файла будут использованы права владельца файла
      • RUID — это настоящий идентификатор пользователя, который не меняется. Когда пользователь входит в систему, оболочка запускается с его реальным идентификатором, и все процессы, запускающиеся из оболочки, наследуют реальный идентификатор.
    • SGID аналогичен SUID, но устанавливаются права группы владельца файла. Если SGID назначен директории, то создаваемые в директории объекты получат идентификатор группы владельца
      • chmod g+s,a+rx <command> - назначить права с SGID
      • EGID (Effective Group ID). То есть процесс выполнится с правами текущего пользователя и группой владельца
    • Sticky bit запрещает удаление файла при наличии достаточных прав, если вы не его владелец. Это правило не распространяется для root.
      • Назначается sticky bit chmod +t

    Access List (ACL)#

    Каждый файл в любой файловой системе будет иметь владельца, группу и права. Когда нескольким пользователям нужен доступ к одному и тому же файлу, а принадлежат они к разным группам, нам помогут расширенные права доступа или Access List (ACL)

    • getfacl - просмотр дополнительных разрешений
      • getfacl test7/file1 > my_acl.txt - сохранить дополнительные права
    • setfacl (setfacl -m u:frontend:rw file1) - назначение дополнительных прав
      • setfacl -m u:frontend:rw,g:root:rwx <file> - одновременно назачить дополнительные права на пользователя и на группу
      • setfacl -m default:u:frontend:rw <dir> - назначить права по умолчанию на директорию
      • setfacl --restore=my_acl.txt - восттановить дополнительные права из файла
      • setfacl -x u:frontend <file> - удаление расширенных прав
      • setfacl -b <file> удаление всех расширенных прав
