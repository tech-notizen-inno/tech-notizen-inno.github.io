    Как работать с дисками#

    • lsblk -f или fdisk -l - посмотеть полный список дисков на ВМ Диски — это блочные устройства: данные разделяются на блоки, и блоки раскладываются по диску. В Linux блочные устройства называются по следующему шаблону: /dev/ + тип устройства + буква диска + номер раздела. Например, /dev/sda1 — первый раздел первого диска типа SCSI.
    • «Тип устройства» — это однобуквенный идентификатор: “h” - IDE дисков, “s” - SCSI и “v” - виртуальных дисков. Каждому новому добавленному диску выделяется следующая буква латинского алфавита. На диске может быть несколько разделов и каждый из них имеет свой номер. Номера разделов начинаются с 1. Например, /dev/sda1 — первый раздел первого диска, /dev/sda2 — второй раздел первого диска, /dev/sdb1 — первый раздел второго диска и так далее. Разметка диска fdisk /dev/vdb - разметка диска. g - создать новый GPT диск, n - создать новую партицию, p - показать размеченную FS, w - записать на. диск partprobe - перечитать информацию о разделах диска после работы fdisk

    Logical Volume Manager#

    LVM управляет тремя сущностями:

    • Физический том — Physical Volume (PV): раздел на физическом диске или весь диск целиком.
      • Создается с помощью утилиты pvcreate <disk1> [<dis2=/dev/vdb2>]
      • Просмотр с помощью pvs и pvdisplay
        • PE Size — это размер чанка. Чанк — это минимальная единица выделения места на диске для LVM.
      • Перемещение тома (при переносе со старого на новый диск): pvmove (pvmove -n my_vol /dev/sda1)
    • Группа физических томов — Volume Group (VG): объединение нескольких физических томов под одним логическим именем.
      • Создадим группу томов lvm (VG) с помощью утилиты vgcreate name disk1 disk2 ... (sausage-store /dev/vdb1 /dev/vdb2)
      • Просмотр с помощью vgs и vgdisplay
      • Расширение группы томов - vgextend (vgextend --verbose sausage-store /dev/vdb3)
    • Логический том — Logical Volume (LV): логический раздел, который можно отформатировать и примонтировать как обычный раздел физического диска.
      • Создать логический том lvcreate -l/-L размер -n имя <название группы> (-l +100%FREE -n logs sausage-store)
      • Просмотр с помощью lvs и lvdisplay
      • Расширение тома lvextend (lvextend /dev/sausage-store/logs /dev/vdb3)
      • Сжатие тома lvreduce (lvreduce -L -500M /dev/sausage-store/logs)
      • Резервное копирование логического раздела - lvcreate -s -n snap -L 5g my_vg/my_vol
      • Откат lvconvert --merge my_vol/snap
      • Удаление lvremove (lvremove /dev/my_vg/my_vol)

    Типы файловых систем#

    Все файловые системы создаются утилитой mkfs.<FS_NAME> (**mkfs.ext4**, **mkfs.xfs**, ..). sudo mkfs.ext4 /dev/sausage-store/logs resize2fs (resize2fs [-f] /dev/sausage-store/logs) - расширить файловую систему Подмонтировать диск $ sudo mkdir -p /var/sausage-store/logs $ sudo mount <устройство> <директория> (/dev/sausage-store/logs /var/sausage-store/logs) Команда mount монтирует диск только до следующей перезагрузки операционной системы. Чтобы диск примонтировался в директорию после перезагрузки хоста, необходимо прописать его в файл /etc/fstab. mount -a - примонтировать записи в fstab unmount (umount /var/sausage-store/logs/) - отмантировать файловую систему. Обратите внимание: в момент, когда мы отмонтируем раздел, не стоит находиться в директории Чек-лист для проверки корректности файловой системы ссылка

    Основные операции#

    1. Создать volume
      1. fdisk - создать физический раздел или несколько разделов
      2. pvcreate - создать физический том (или несколько томов)
      3. vgcreate - создалть группу томов
      4. lvcreate - создать volume
      5. mkfs.extXXX - создать файловую систему
    2. Добавить диск в volume
      1. если нужно - fdisk - создать раздел
      2. pvcreate новые физический том
      3. vgextend - расширить групу томов
      4. lvextend - расширить логический том (volume)
      5. resize2fs - расширить файловую систему
    3. Уменьшить volume
      1. unmount - отмонтировать файловую систему
      2. fsck.ext4 (fsck.ext4 -f /dev/sausage-store/logs) - проверить файловую систему на ошибки
      3. lvreduce - сжать том
      4. resize2fs - сжать файловую систему

    Дисковые утилиты#

    • dd (dd if=/dev/zero of=/var/sausage-store/logs/payload bs=1M count=2048) - convert and copy a file
    • df (df -h) - report file system disk space usage
        • df -i - посмотреть статистику использования индексных дескрипторов можно командой . В выводе будут столбцы: Inodes, IUsed, IFree, IUse%
    • du (du -h / du -h -d 1 /run) - используемое место на дисках (рекурсивно)
    • ncdu (ncdu /run) - демонстрация файлов в виде псевдографики
    • lsof (lsof -u frontend -a /run) - отображениие файлов процессов
      • lsof -a +L1 /run покажет только удалённые файлы в конкретной директории, которые занимают память из-за того, что процесс не отпустил дескрипторы
    • ls -li - первый символ указывает на тип файла
    • ls -la - второй столбец количество ссылок на файловый дескриптор
    • ln (ln -s /home/student/file1 link2) - создание ссылки (-s (soft) - мягкой). Жесткиие ссылки - ссылаются на файловый дескриптор также как и файл, Мягкие содержать путь. Файл не будет удален если на него есть жесткая ссылка.
    • stat (stat file1) - просмотр файлового дескриптора (inode)
      • inode содержит { владелец, права,размер, время последней модификации,время создания, группа, местонахождение файла}
      • find . -inum 272274 - поиск по файловому дескриптору
    • mkfifo или mknod <name> p - создать именованный канал . В Linux именованный канал отображается как файл с типом p и размером 0. Создав такой канал, мы также можем перенаправить в него вывод других команд, но в отличие от неименованного канала, перенаправление вывода не завершится и надо будет выполнить прерывание Ctrl+c
