    Команды для работы с Systemd#

    systemctl list-units --type service --all — просмотр всех юнитов в системе systemctl start name — запустить сервис systemctl stop name — остановить сервис systemctl restart name — перезапустить сервис systemctl status name — посмотреть статус сервиса systemctl reload name — перечитать конфигурацию systemctl daemon-reload — перечитать конфигурацию для всех systemctl try-restart name — перезапустить, если запущен systemctl enable name — включить автозапуск сервиса systemctl disable name — отключить автозапуск сервиса systemctl list-unit-files --type service — список установленных юнит-файлов сервисов В зависимости от дистрибутива, в директории /lib/systemd/system/, /usr/lib/systemd/user/, /usr/lib/systemd/system/. Есть ещё директория /etc/systemd/system/ — место для дополнительных юнит-файлов, сюда мы также будем добавлять свои

    Формат файлов-юнит#

    Директива OnCalendar указывает время или частоту запуска таймера. В общем случае формат директивы такой: "<дни недели> <год>-<месяц>-<числа месяца> <часы>:<минуты>:<секунды>". Дни недели необязательны и в нашем примере значение "OnCalendar=*-*-* *:*:00" читается как «каждую нулевую секунду» (по-русски — каждую минуту). systemd-analyze calendar "*-*-* *:*:00" - проверка календаря
