    Основные сущности#

    Ansible Task#

    Ansible Task — это блок в YAML, с помощью которого можно сообщить Ansible, что мы ожидаем получить на целевом сервере: установленный пакет, желаемую конфигурацию и запущенный сервис.

    • Список встроенных модулей Ansible (Ссылка)
    • Список всех модулей Ansible (Ссылка)

    Ansible Role#

    Ansible-роль представляет собой директорию, с упорядоченными по назначению YAML-файлами.

    Ansible Play#

    Ansible Play — это блок описания в YAML, в котором указан шаблон целевых хостов (или ещё говорят host pattern) и список задач или ролей для этих хостов.

    Ansible Playbook#

    Список плеев организуется в Ansible-плейбук (playbook) — YAML-файл, который передаётся утилите ansible-playbook

    • ansible-playbook playbook.yaml - запустить playbook
      • ansible-playbook --private-key ./vault/tf-ansible  playbook.yaml - с указанием приватного ключа
    • --check, --diff, --list-hosts, --list-tasks, and --syntax-check - проверка playbook
    • -vvv - подробная отладочная информация

    Inventory#

    Конфигурация, необходимая Ansible для подключения к целевым серверам, называется inventory

    Структура проекта Ansible#

    • Для организации репозиториев с Ansible кодом используют коллекции. Коллекция инкапсулирует весь необходимый код для какой-то высокоуровневой задачи.
    • Итоговый воркфлоу автоматизации управления конфигурацией может выглядеть следующим образом:
      • Описание зависимости в специальном файле (по соглашению requirements.yaml), в котором указываете список используемых коллекций.
      • Устанавливаются зависимости с помощью утилиты ansible-galaxy и формируете inventory.
      • Run ansible-playbook с указанием плейбука.

    Переменные Ansible#

    Статические переменные задаются в group_vars Переопределение переменных (Порядок)

    1. ключ ansible-playbook --extra-vars "my_var=value"
    2. group_vars/<my group>.yaml
    3. Ansible-inventory
    4. Ansible-roles

    Ansible Facts#

    При запуске плея, при первом подключении к хостам и до запуска указанных задач, Ansible запускает модуль setup, который выполняет анализ окружения целевого хоста и формирует набор переменных в словаре ansible_facts. Отключить сбор фактов Получение только определенного набора фактов Кэширование фактов через ansible.cfg

    Конфигурация Ansible#

    • Прописывается в файле ansible.cfg
    • Конфигурационый файл может быть расположен
        1. Путь из переменной окружения ANSIBLE_CONFIG.
        1. ./ansible.cfg (файл в текущем каталоге).
        1. ~/.ansible.cfg (файл в домашнем каталоге).
        1. /etc/ansible/ansible.cfg.
    • ansible-config init --disabled -t all > ansible.cfg - сгенерировать конфиг (работает в последних версиях)
    • Ссылка на пример конфига в 2.10

    Ansible Vault#

    Для передачи чувствительных параметров их можно шифровать и при запуске playbook передать пароль для расшифровки

    • зашифровать данные ansible-vault encrypt_string "my_password" --name the_secret --vault-password-file=./vault/.vault
      • name - имя переменной
      • vault-password-file - файл содержащий пароль
    • в файле переменных указать
    • запросить запрос пароля при проигрывании playbook ansible-playbook playbook.yaml --ask-vault-pass
    • либо сформировать файл с паролем и указать его в ansible.cfg
    • проверить зашифрованое через decrypt (сформировать файл + вызвать decrypt)

    Основные команды#

    ansible-galaxy init <my role name> - сгенерировать ansible-role ansible-playbook playbook.yaml - запуск playbook¶ ansible-playbook playbook.yaml --limit dev - запуск playbook на определенном окружении ansible <IP> -m setup - Запустите ad-hoc команду для получения переменных ansible_facts ansible-config - покажет список всех параметров конфигурации, их значения по умолчанию
