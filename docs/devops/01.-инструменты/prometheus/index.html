    В метрике гистограммы также экспортируются два счётчика: сумма всех измерений (sum) и общее количество измерений (count), что позволяет посчитать простое среднее. 

    # HELP prometheus_interval_length_seconds Actual intervals between scrapes.

    PromQL — язык запросов Prometheus#

    • http_requests_total finstance="frontend", env="test"｝- временной ряд связанный с метрикой http_requests_total и значениями лейблов в фигурных скобках
    • http_server_requests_seconds_count{method="GET",uri="/actuator/prometheus"}[5m] - временной ряд можно указывать в квадратных скобках
    • rate(http_server_requests_seconds_count{method="GET",uri="/actuator/prometheus"}[5m]) - позволяет посчитать скорость возрастания счётчика во временном промежутке
      • rate by (endpoint) - скорость по лейблам

    Агрегирующие функции#

    Формат задания функции

    • sum(rate(http_server_requests_seconds_count[5m])) - функция sum выдает сумму всех скоростей (rate) роста метрики http_server_requests_seconds_count
    • sum by (uri) - cумма с группировкой по лейблу
    • histogram_quantile - принимает на вход значение интересующего нас квантиля и имя временных рядов разбиений гистограммы. Пример histogram_quantile(0.99, rate(http_server_requests_seconds_bucket[5m]))
    • offset - позволяет получить прошлые данные. Например, на 1 день назад http_requests_total offset 1d
