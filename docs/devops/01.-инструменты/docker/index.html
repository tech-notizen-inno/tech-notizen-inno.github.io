    Образы#

    Docker очень рационально использует место. Образ состоит из слоёв, каждый из которых представляет неизменяемую файловую систему, то есть набор файлов и папок. В целом образ является объединенной файловой системой (Union File System, или Union FS). Каждый слой в Docker может быть только для чтения, за исключением верхнего, который поддерживает изменения. В .dockerignore можно задавать шаблоны каталогов и файлов, которые попадут в образ. многоэтапная (multi-stage) сборка

    Ключевые слова Dockerfile#

    Для запуска процесса в контейнере обычно используют RUN, ENTRYPOINT или CMD. RUN хорош для служебных действий вроде установки пакетов или смены каталога. Чтобы запустить основной процесс контейнера, используют ENTRYPOINT или CMD, без них контейнер не взлетит. Основная разница между ними в том, что при использовании CMD параметры командной строки, которые вы передаёте в момент запуска контейнера в командной строке, могут перезаписать параметры, указанные в Dockerfile. В случае с ENTRYPOINT, параметры из командной строки не могут перезаписать значения из Dockerfile

    Команды#

    !!! ==Пользователь, который применяет команды должен быть в группе docker==

    • docker run <адрес репозитория>/<имя репозитория>/<имя образа>:<тег образа> <команда, которую нужно выполнить внутри контенера> - запустить контейнер
      • docker run --rm -d --name sausage-backend --env-file .env --network=docker_network (название образа):(tag)
      • -d - запуск в detached режиме
      • -rm - автоматическое удаление контейнера после запуска
      • -p 9999:8080 - проброс портов
      • -v ~/caddy/conf:/caddy - подмантировать ~/caddy/conf папку в образ
      • --env-file - запуск с переменными окружения из файла
      • --network=sausage_network - указать при запуске сеть
      • --restart=on-failure:10 - рестартовать при падении
      • docker run -it --log-driver json-file --log-opt max-size=10m --log-opt max-file=3 - задать опции файлов логов
    • docker ps -a - посмотреть работающие и остановленные контейнеры
    • docker images - посмотреть список образов в локальном репозитории
    • docker rm - удалить контейнер, опция -f - принудительное удалениие
    • docker exec -it hello bash - выполнить команду bash в контейнере hello; i - interactive, t - terminal
    • docker commit (название контейнера) (название комита) - создает новый образ на основе изменений в контейнере
    • docker start (id контейнера) - стартануть контейнер
    • docker stop (id контейнера) - остановить контейнер
    • docker version - узнать версию Docker
    • docker pull , docker pull tennyy/postman - обновить образ (или создать новый), скачивание своего образа с DockerHub
    • docker push <dockerhub username>/(название образа) - загрузить образ в репозиторий
    • docker login - авторизация в DockerHub
    • docker logout - выйти из авторизованного пользователя
    • docker attach (id контейнера) - выйти из интерактивного режима
    • docker build -t (название образа) . - собрать образ в текущей директории,
      • docker bild -t (название образа):version - добавить тег образу
      • docker build --no-cache - чтобы образ пересобрался полностью, так как Docker кэширует слои при сборке
      • docker build -t sausage-backend:$VERSION --build-arg VERSION=$VERSION . - сборка и передача аргумента
    • docker logs (id/name контейнера) - посмотреть логи контейнера, опция -f показывать логи в реальном времени
    • docker tag (старое название образа) (новое название) - смена названия образа, посредством создания нового
    • docker cp (название файла) (название контейнера):(путь в контейнере) - добавить файлы в контейнер
    • docker inspect (id контейнера) - Return low-level information on Docker objects.
    • docker rmi (image) - удалить образ
    • docker network create -d bridge sausage_network - создать сеть для взаимодействия контейнеров
    • docker container ls --filter health=healthy - вывести список “здоровых” контейнеров
    • docker --context remote compose --env-file deploy.env up backend -d --pull "always" --force-recreate - запуск контейнеров при помощи context на удаленном сервере
      • --context remote: указывает на удалённый контекст Docker, который определён заранее. Контекст представляет собой среду, в которой выполняются команды Docker.
      • up <service>: запускает сервис, указанные в файле docker-compose.yml
      • --pull "always": указывает Docker Compose всегда обновлять образы контейнеров до последней версии при запуске.
      • --force-recreate: принудительно пересоздаёт контейнеры, даже если они уже существуют.
    • docker context ls - посмотреть все контексты
    • docker inspect - можно использовать, чтобы посмотреть переменные среды
    • docker exec <container> env - посмотреть переменные внутри контейнера
    • docker network ls - посмотреть сети
    • docker volumes ls - посмотреть подключаемые тома
    • удалить ненужные образы (в примере по ключу praktikum) - `docker rmi -f docker images | grep praktikum | awk '{ print $3}' | tr '\n' ' '

    Docker Compose Command#

    • docker compose up -d --scale web=5 - запуск со скалированием контейнеров
    • docker compose top - top для контейнерова в docker-compose
    • docker compose events - отображение событий в контейнерах
    • docker compose -f docker-compose-minio.yml up -d - запуск с именем файла
    • docker compose --profile <profile-name> up - запуск контейнеров с указанным профайлом
      • docker compose --profile "*" up - запуск со всеми указанными профайлами

    Health Check#

    В Dockerfile HEALTHCHECK CMD curl --fail -s http://localhost:8080/<health check endpoint>

    В docker compose healthcheck: test: curl --fail -s http://localhost:8080/<health check endpoint> interval: 30s timeout: 10s start_period: 20s retries: 6

    Статус для контейнеров с healthcheck отображает состояние “здоровья”

    curl может быть заменен на wget --no-verbose --tries=1 --spider http://localhost || exit 1
