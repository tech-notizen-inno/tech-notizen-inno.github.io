    Миграции#

    Инструменты миграции - Flyway или Liquibase.

    Flyway#

    • migrate — старт миграции, запустятся все скрипты миграции которые находятся в папке migration
    • clean — запустятся скрипты очистки базы (которые надо самостоятельно прописать)
    • info — даст информацию о статусе об уже прошедших миграциях и тех, что можно провести сейчас базируясь на таблице flyway_schema_history
    • validate — запустится встроенная проверка на корректность синтаксиса SQL-скриптов

    Миграционные скрипты подразделяются на типы:

    • Versioned Migrations — основные скрипты миграции, в которых создаются таблицы, схемы, пользователи, вносятся тестовые данные. Начинаются с V001__.
    • Undo Migrations — скрипты, предназначенные для отмены миграции. Начинаются с U01__.
    • Repeatable Migrations — скрипты, предназначенные для повтора и выполняемые каждый раз при запуске приложения. Используются для выдачи прав пользователям, создание VIEW-шек, функций и много другого. Запускаются последними, после того как отработали скрипты миграции. Начинаются с R__.

    Адрес подключения к базе данных и расположение скриптов миграции указываются в конфигурационном файле flyway.conf.

    Способы использования#

    • Независимая утилита. Пример запуска flyway flyway -user=my_sql -password=23e2f345f3 -locations='filesystem:db/migration' -url="jdbc:sqlserver://192.168.1.30;database=test" -target=4 migrate
    • Встраивается через pom.xml. пример
    • Запуск из Java приложения
    import org.flywaydb.core.Flyway;`

public class App {
    public static void main(String[] args) {
        Flyway flyway = Flyway.configure()
				        .dataSource(
					        "jdbc:h2:file:./target/foobar", "sa", null)
					        .load();
        flyway.migrate(); // Здесь запускается миграция
    }
}
    • Отключение миграции через переменную среды в Spring приложениии SPRING_FLYWAY_ENABLED=false

    Mongo#

    Основные команды mongosh

    • mongosh --norc --tls --tlsCAFile /home/student/.mongodb/root.crt --host 'rs01/rc1a-gck90cwqmi3ls3y5.mdb.yandexcloud.net:27018' --username std-028-50 --password Testusr1234 std-028-50 - подключение к БД std-028-50 по TLS с сертификатом
    • db - посмотреть базы
    • show collections - список коллекций
    • db.stats() - статистика базы
    • db.xxxx - команды для работы с БД
      • db.products.find() - посмотреть значение коллекции products (имя соответствует названию коллекции)
      • db.products.find({price: {$lt: 60}}) - поиск конкретного значения
      • db.products.insertOne({name: "Hot Dog", price: 70.00}); - вставить запись (JSON) в колелкцию products

    Добавить mongo в проект с Java (pom.xml)

    Spring Boot позволяет все переменные, заданные в src/main/resources/application.properties, переопределить с помощью переменных окружения, используя заглавные буквы и заменив точки на символ подчёркивания.

    Например, параметр spring.data.mongodb.uri можно переопределить используя переменную окружения: SPRING_DATA_MONGODB_URI={строка подключения}.
