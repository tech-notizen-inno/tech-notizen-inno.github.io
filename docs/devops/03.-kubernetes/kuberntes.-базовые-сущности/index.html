    Pod#

    Один Pod представляет собой один инстанс приложения, включающий один или несколько Docker-контейнеров.

    Дополнительные виды контейнеров#

    Манифест для Pod#

    В блоке spec указывают специфичные для конкретной сущности атрибуты. Для Pod’а такими атрибутами будут: контейнеры, которые мы хотим запустить в нём, и образы, на основе которых будут создаваться эти контейнеры. imagePullPolicy определяет для kubelet политику скачивания образа контейнера: - IfNotPresent — скачивать образ контейнера, если отсутствует на Node. - Always — скачивать образ каждый раз, когда kubelet запускает контейнер. - Never — использовать только те образы, которые присутствуют на Node; если образа нет — контейнер не запустится. Чтобы контейнеры появились на Node, их нужно туда «скачать» самостоятельно.

    Probes#

    • livenessProbe — запущен ли контейнер?
      • HTTP GET (200, 300) - провека успешная
      • TCP Socket - проверка на подключение к сокету
      • Exec - выполнение произвольной команды внутри контйнера (результат=0 - успех)
      • Параметры
        • initialDelaySeconds — через сколько секунд после старта контейнера начать проверки.
    • startupProbe — запущено ли приложение внутри контейнера?
      • Используется когда нужно прогреть кэши или накатить миграции на базу данных. В этом случае используется startupProbe. Синтаксис у startupProbe такой же, как и у остальных проверок. Если указана startupProbe, то запуск livenessProbe и readinessProbe блокируется до тех пор, пока не пройдет startupProbe.
    • readinessProbe — приложение готово работать с пользователями и можно ли пускать трафик? Описание проб .
