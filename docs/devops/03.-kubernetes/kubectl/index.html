    Шпаргалка Общие команды

    • kubectl get <ресурс> - получить информацию о ресурсе
      • kubectl get nodes --output wide - расширенная информация о нодах
      • kubectl get nodes --show-labels - информация о нодах, включая метки
      • kubectl get pods --watch - отслеживание изменения в подах
      • kubectl get services - информация о сервисах
      • kubectl get endpoints nginx - информация об endpoint nginx
      • kubectl get persistentvolume - информация о persistentvolume
      • kubectl get job - вывести все jobs
      • Kubectl get jobs.batch - вывести все jobs, собранные в batch
      • kubectl get cronjobs.batch - - вывести все cronjobs
      • kubectl get svc/kube-dns -n kube-system - список сервисов DNS
      • kubectl get endpoints/kube-dns -n kube-system - список endpoint
      • kubectl get secrets - информация о секретах
      • kubectl get secrets nginx -o jsonpath='{.data}' - выгрузить секреты в json в формате base64. Посмотрет в текстовом виде echo 'cG9zdGdyZXM=^| base64 --decode
      • kubectl get rc - Посмотреть контроллеры репликации через kubectl
      • kubectl get horizontalpodautoscaler - список HPA
      • kubect1 get all - выдаст все используемые нами компоненты
    • kubectl api-resources - отображает ресурсы API, краткие и длинные имена
    • kubectl explain <сущность> - отображает верхнеуровневую документацию в консоли
    • kubectl describe <ресурс> - показать статус ресурса
      • kubectl describe pod nginx - показать статус pod с имененем nginx
      • kubectl describe nodes - показат статус нод
      • kubectl describe services nginx - показывает статус сервисов
      • kubectl describe secrets nginx - показывает секреты на nginx
      • kubectl describe job hello-with-timeout - показывает job c именем hello-with-timeout
    • kubectl apply -f pod.yaml - создать или изменит ресурс используя манифест (kubectl create - только создать, kubectl delete - удалить сущность)
      • kubectl apply -f backend - применит все файлы из папки backend
      • kubectl delete pod <name> - удалить под
      • kubectl delete rc backend - удалить replication controller
      • kubectl delete rc backend --cascade=false - удалить ReplicationController, но оставить Ро’ы,
    • kubectl logs - показать логи (kubectl logs nginx - показат логи nginx)
      • аргументы: --tail 10, - f, --since=1 (подробности можно посмотреть в справке kubectl logs --help)
      • kubectl logs <имя Pod'а > --previous - посмотреть логи упавшего пода
    • kubectl exec -it nginx -- bash - исполнит в контейнере nginx команду bash
    • kubectl edit <type> <name> - редактировать кластер “налету” по типу и сущности (например, kubectl edit pod backend - редактивание pod backend)
    • kubectl port-forward nginx 8080:80 - проксирование портов для pod
    • kubectl port-forward service/nginx 8080:80 - проксирование портов для сервисов
    • kubectl scale rc backend --replicas=5 - изменить число реплик
    • kubectl create secret generic nginx --from-literal=dbname=postgres --from-literal=host=db-postgre --from-literal=password=QWERTY123 --from-literal=username=postgres - создать секреты
    • kubectl label nodes cl17it1ti54cda2q2d0h-evow cpu=highPerformance - установить метку на ноду
    • kubectl get --raw='/readyz', kubectl get --raw='/ readyz?verbose' - прямой вызов API

    Работа с деплоем#

    • kubectl rollout history deployment/backend-report - просмотр версионирования деплоя
    • kubectl rollout status deployment/backend-report - просмотр статуса деплоя
    • kubectl rollout undo deployment/backend-report --to-revision=2 - откатиться к ревизии
    • kubectl rollout pause deployment/backend-report - остановить процессы rollout
    • kubectl rollout resume deployment/backend-report - возобновить процессы rollout
    • kubectl autoscale deployment/backend-report --min=3 --max=10 --cpu-percent=80 - создать HPA

    Работа с контекстом#

    • kubectl config current-context - текущий контекст
    • kubectl config get-contexts minikube - конфигурацию контекста minikube
    • $ kubectl config use-context <contextname> --kubeconfig=<your_kubeconfig_file> - переключение контекста
    • kubectl config view --minify - посмотреть текущий kubeconfig

    Флаги kubectl#

    • -o yaml - вывод в формате yaml
    • -o json - вывод в формате json
