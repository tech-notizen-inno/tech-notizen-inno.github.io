    Релизы#

    Релиз — это экземпляр чарта, запущенный в Kubernetes. Для хранения информации о релизе Helm использует secrets или configmap, содержащий в разделе data.release (данные зашифрованы в base64): - имя релиза - ﻿﻿информация о релизе: статус и его словесное описание, время создания и
    обновления - ﻿﻿чарт: метаданные ( Chart.yaml ), шаблоны (в исходном виде) и значения переменных для этого релиза (не только дефолтные, но и те, которые мы переопределили) - ﻿﻿манифест (отрендеренные шаблоны) - ﻿﻿версия приложения - ﻿﻿namespace, куда задеплоен чарт

    Чтобы выпустить новый релиз, нам нужно сделать хотя бы одну из 2 вещей, но можно и обе:

    1. ﻿﻿﻿Поменять в Chart.yaml значение y version.
    2. ﻿﻿﻿Внести какие-то изменения в шаблоны/

    Команды (ссылка на документацию)#

    • helm list - посмотреть установленные релизы и их время жизни
    • helm lint ./<chart_name> - прогнать через linter
    • helm template ./<chart_name> - сгенерировать манифест и вывести в stdout
      • helm template ./<chart_name> -f ./path_to_value/values.yaml --debug - генерация манифеста в дебаг режиме используею заданный value файл

    Работа с репозиториями#

    • helm repo list - список репозиториев
    • $ helm repo add <helm_repository_name> http://<host>:<port>/repository/<nexus_repository_name>/ --username <username> --password <password> - подключить новый репозиторий
    • helm repo update - обновить информацию о чартах в репозиториях
      • helm repo update <helm_repository_name> - обновит информацию о чартах в репозитории для репозитория <helm_repository_name>
    • helm search repo - поиск по всем пакетам (--version для поиска конкретной версии, можно указать `<helm_repository_name> для поиска в конкретном репозитории)

    Работа с релизами (чартами)#

    • helm create <имя чарта> - создать структуру для чарта
    • helm package ./<chart_name> - собрат пакет с именем Chart. Name-Chart.Version.tgz
    • helm install ‹name> ‹chart_name>, - установка пакета
      • helm install < helm_repository_name> <chart_name> --version «chart_version> - установка пакета определенной версии
      • helm install ‹name> ./<chart_name> - установка пакета локально из папки
      • helm install --set image.tag=1.21.5 nginx ./nginx-test - установить пакет nginx переопределив переменную image.tag
    • helm upgrade ‹name> ‹helm_repository_name>/<chart_name> - обновление пакета
      • helm upgrade ‹name> ./<chart_name> - обновление пакета из локальной папки
      • Чтобы в случае ошибки upgrade Helm автоматически откатил релиз на последний успешный, используем ключ --atomic.
    • helm uninstall <name> - удалить релиз
      • --keep-history - история останется и можно будет откатить релиз
    • helm history <nginx> - показать историю релизов чарта nginx
    • helm rollback ‹name> revision_number - откатиться на указанный релиз
      • helm rollback nginx 1 - откатить чарт nginx к 1-му релизу
    • helm get all <sausage-store> - отобразить состав всего релиза
