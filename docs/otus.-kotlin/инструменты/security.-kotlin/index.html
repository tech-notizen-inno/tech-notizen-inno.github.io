    Виды уязвимостей#

    SQL Иньекции
    *Решение. Использовать специальные классы, проверяющие параметры *
    XSS-Иньекции
    Внедрение в параметры запроса скриптов для исполнения в браузере жертвы vulnsite.com/search/?q=test<script>alert(%27XSS%27);</script>
    Решение. Экранировать все “опасные” символы-разделители HTML
    CSRF (Cross Site Request Forgery)
    Решение. Предотвращение CSRF требует включения непредсказуемого токена в каждый НТТР-запрос. Предпочтительный вариант - включить уникальный токен в скрытое поле
    *Решение. Ktor имеет встроенную функцию защиты от CSRF (https://youtrack.jetbrains.com/issue/KTOR-2910/CSRF-protection-feature) *
