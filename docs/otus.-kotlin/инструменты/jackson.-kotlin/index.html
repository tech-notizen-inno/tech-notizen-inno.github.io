    Jackson. Kotlin

    • Примеры сериализации и десериализации
    • Аннотоации в Jackson

    Позволяет преобразовывать JSON в классы Java/Kotlin ObjectMapper — наиболее часто используемая часть библиотеки Jackson, так как является самым простым способом преобразования между POJO и JSON. Она находится в com.fasterxml.jackson.databind.

    Примеры сериализации и десериализации#

    • String json = objectMapper.writeValueAsString(employee) - Записть в JSON
    • Employee employee = objectMapper.readValue(employeeJson, Employee.class)- Создание объекта
    • `List employeeList = objectMapper.readValue(file, new TypeReference<>(){}); - Создание массива объектов
    • Map<String, Object> employee = objectMapper.readValue(file, new TypeReference<>(){}); - Создание map из JSON
    • objectMapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false); - Конфигурирование на пропуск неизвестных полей
    • mapper.enable(SerializationFeature.INDENT_OUTPUT); - Использовать отступы при сериализации

    Аннотоации в Jackson#

    Пример использования аннотаций |

    Аннотация Объяснение Пример
    @JsonFormat Задание формата даты @JsonFormat(pattern = "dd/MM/yyyy")
    @JsonSetter Сопоставить поле в строке JSON с полем в POJO, где их имена не совпадают.
    @JsonAnySetter Эта аннотация полезна в случаях, когда JSON содержит некоторые поля, не объявленные в POJO.
    @JsonGetter Полезно, когда мы хотим сопоставить поле POJO с полем JSON, используя другое имя.
    @JsonAnyGetter Эта аннотация позволяет нам использовать объект Map как источник свойств JSON. Скажем, у нас есть эта карта как поле в классе Cat.
    @JsonIgnore Поле с аннотацией игнорируется как при записи, так и при чтении JSON.
    @JsonProperty Позволяет изменить имя свойства, в которое будет сериализировано поле. Еще этой аннотацией можно помечать методы, и их возвращаемое значение будет преобразовано в JSON-свойство при сериализации:
    @JsonInclude С помощью этой аннотации можно указать, в каком случае поле должно сериализироваться.
    @JsonPropertyOrder позволяет задать порядок сериализации полей
