    Companion#

    Companion. Позволяет делать некоторые члены класса быть общими для всех объектов

    • Можно объявить объект-singleton внутри класса, используя ключевое слово companion
    • Использовать члены companion-объекта можно, используя имя класса в качестве квалификатора

    Typealias#

    Generic. Ковариантность и контрвариантность#

    Inline/value-класс

    • Обертка над типом, при этом должны иметь единственное свойство, инициализируемое в основном конструкторе
    • ﻿﻿Основная идея - конкретизировать контекст использвольвания того или иного типа, не теряя производительность примитивных типов
    • ﻿﻿Позволяет добавлять свойства (динамические) и методы
    • ﻿﻿Оптимизирует производительность используя boxing/unboxing
    • ﻿﻿Не может наследовать другой класс, но может реализовать интерфейсы

    Делегирование#

    Делегирование интерфейса
    Делегирование свойств
    • Переопределение в объекте, которые делегирует другому управление позволяет “перекрыть” делегируемую функцию
    • Методы делегирующего класса НЕ будут вызываться в делегируемом, так как он будет скомпилировать “не зная” о переопределении
    • Повторное использование общего поведения различных свойств - к примеру, ленивая инициализация, вызов кода, при изменении свойств или же хранение данных в Мар вместо отдельных полей
    • ﻿﻿Используете ключевое слов by => получаете delegated properties
    • ﻿﻿Вызовы get делегируется оператору getValue объекта-делеагата
    • Вызов set делегируется оператору setValue объекта-делегата
