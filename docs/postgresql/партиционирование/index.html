    Виды партиционирования (секционирования)#

    Секционирование по диапазонам - Таблица секционируется по «диапазонам» определённым по ключевому столбцу или набору столбцов и не пересекающимся друг с другом. Секционирование по списку - Таблица секционируется с помощью списка, явно указывающего, какие значения ключа должны относиться к каждой секции Секционирование по хешу - Секции содержат строки, для которых хеш-значение ключа, делённое на модуль, равняется заданному остатку.

    • В секции хранится подмножество строк таблицы, заданное границами секции.
    • Строки, вставляемые в секционированную таблицу, перенаправляются в соответствующие секции в зависимости от значений столбцов ключа разбиения.
    • Если при изменении значений ключа разбиения в строке она перестаёт удовлетворять ограничениям исходной секции, эта строка перемещается в другую секцию (начиная с 11 версии).

    Как секционировать таблицу#

    • Декларация секционирования состоит из определения метода разбиения и списка столбцов или выражений, образующих ключ разбиения.
    • Секционированная таблица является «виртуальной» и явно не хранится.
    • Секции являются обычными таблицами, связанными с секционированной, они и хранятся.
    • В секции хранится подмножество строк таблицы, заданное границами секции
    • Строки, вставляемые в секционированную таблицу, перенаправляются в соответствующие секции в зависимости от значений столбцов ключа разбиения
    • Если при изменении значений ключа разбиения строка перестаёт удовлетворять ограничениям исходной секции, она перемещается в другую секцию
    • Удаление секции DROP TABLE.
    • Отключение секции с сохранением данных ALTER TABLE DETACH PARTITION.
    • Добавление секции CREATE TABLE ... PARTITION OF ... FOR VALUES.
    • Раздел по умолчанию может хранить строки, которые не попадают в диапазон или список существующих разделов. Синтаксис CREATE TABLE ... PARTITION OF ... DEFAULT;

    Преобразование таблицы в секционированную#

    Вариант 1. Секционирование существующей таблицы по диапазону#

    1. Выбор ключа секционирования
    2. Определение диапазона значений ключа
    3. Переименование исходной таблицы (простой)
    4. Создание секционированной таблицы
    5. Добавление ограничения CHECK на исходную таблицу без проверки
    6. Изменения в словаре значения проверки на истину
    7. Прикрепление исходной таблицы как секции в новую
    8. Создание остальных секций, секции по умолчанию
    9. Удаление PK на исходной таблице
    10. Фиксация изменений

    Вариант 2. Секционирование существующей таблицы и распределение данных по разным секциям#

    1. Убедиться, что у исходной таблицы есть первичный ключ
    2. Создать секционированную таблицу
    3. Создать скрипт переноса, который будет пачками переносить строки из исходной таблицы (получить следующие 1000 строк исходной таблицы; вставить эти строки в секционированную)
    4. Создать триггер, который будет обновлять строки в секционированной таблице. (При вставке/обновлении строки вставить/обновить соответствующую строку в секционированной таблице. При удалении — удалить. Если ничего не обновилось или не удалилось, ничего страшного).

    Так как триггер выполняется в одной транзакции с DML-запросом, то на исходную строку будет наложена исключительная блокировка, т.е. все дальнейшие модификации этой строки будут приостановлены до окончания работы транзакции

    1. Необходимо учитывать возможность обновления первичного ключа в исходной таблице и тот случай, когда строки в исходной таблице ссылаются друг на друга (дерево, список и т. п.). В таком случае соответствующие ограничения надо будет накладывать после окончания работы скрипта

    2. После окончания работы скрипта из п. 3 вы получите актуальную секционированную копию исходной таблицы, после чего их можно соответствующим образом переименовать (вызывает простой)
