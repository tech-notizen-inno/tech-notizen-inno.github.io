    Составные типы#

    • набор именнованных атрибутов (полей)
    • создание таблицы вводит одноимённый составной тип
    • CREATE TYPE имя AS (атрибут [, атрибут...] ); - объявление
    • (значение [, значение...])' или ROW (значение [, значение...]) - создание

    Анонимные блоки#

    DO [ LANGUAGE имя_языка ] код: - объявление

    • не принимает и не возвращает значение
    • не сохраняется в БД

    Функции#

    • [IN/OUT/INOUT] <Имя> тип [{DEFAULT=}<Значение>] - переменная функции
    • BEGIN - не должен содержать после себя “;”, иначе он будет интерпретироваться как начало транзакции
    • Категории изменчивости
      • Volatile - Возвращаемое значение может произвольно меняться при одинаковых значениях входных параметров.
      • Stable - Значение не меняется в пределах одного оператора SQL. Функция не может менять состояние базы данных.
      • Immutable - Значение не меняется. Функция детерминирована. Функция не может менять состояние базы данных.
    • является объектом БД
    • возвращают одно значение;
    • обычно вызываются в списке выборки запроса;
    • при вызове во фразе FROM возвращают одну строку;
    • синтаксически таблица. столбец = столбец(таблица), что делает функции и столбцы «взаимозаменяемыми».
    • ALTER FUNCTION — изменение свойств функции ;
    • DROP FUNCTION - удаление функции .

    Объявление переменных#

    • область видимости переменной - текущий блок, включая вложенные блоки и секцию обработки ошибок.

    Табличные функции#

    RETURNS SETOF тип, RETURNS TABLE( атрибут [, атрибут...] ):

    • могут возвращать несколько строк;
    • обычно вызываются в предложении FROM;
    • можно использовать как представление с параметрами, особенно в сочетании с подстановкой тела функции в запрос.

    Команды без результата#

    • Команды SQL встраиваются в код PL/pgSQL
    • одна строка результата
    • RETURN QUERY запрос;
    • RETURN NEXT [выражение];

    Курсоры#

    • Курсор подразумевает итеративную обработку
      • полная выборка занимает слишком много памяти;
      • нужна не вся выборка, но размер заранее неизвестен;
      • способ отдать управление выборкой клиенту;
      • действительно требуется построчная обработка (обычно нет).
    • объявление переменной - курсора (явное), также можно использовать (refcursor)
    DECLARE
	key integer;
	curs4 CURSOR FOR SELECT * FROM tenk1 WHERE unique1 = key; 
BEGIN
	key := 42;
	OPEN curs4;
    • использование функции с курсором
    CREATE TABLE test (coltext); 
INSERT INTO test VALUES ('123');

CREATE FUNCTION reffunc(refcursor) RETURNS refcursor 
AS $$
BEGIN
	OPEN $1 FOR SELECT col FROM test;
	RETURN $1;
END;
$$ LANGUAGE plpgsql;

BEGIN; -- транзакция
SELECT reffunc('funccursor');
FETCH ALL IN funccursor;
COMMIT;
    • цикл по курсору и запросу
    -- цикл LOOP по курсору
OPEN курсорная_переменная FOR запрос;
LOOP
	FETCH курсорная_переменная INTO цель;
	EXIT WHEN NOT FOUND;
	тело-цикла
END LOOP;
CLOSE курсорная_переменная;

-- цикл FOR по курсору
FOR запись IN курсорная_переменная LOOP 
	тело-цикла
END LOOP;
-- цикл по запросу
FOR цель IN запрос LOOP 
	тело-цикла
END LOOP;
    • операции с курсором
      • выборка:
        • FETCH INTO цель; цель может быть переменной составного типа или списком скалярных переменных.
        • только по одной строке, проверка - переменная FOUND;
      • обновление или удаление текущей строки:
        • WHERE CURRENT OF;
        • возможно только для простых запросов (одна таблица, без группировок и сортировок).
      • закрытие:
        • явное командой CLOSE;
        • автоматическое при завершении

    Динамические команды#

    • конструкция USING - подстановка значений параметров
    • format(’%I’), quote_ident - подстановка имено объектов
    • format(’%L’), quote_literal, quote_nullable - подстановка литеролов
    • экранирование строк: $$ ... $$

    Массивы#

    • объявление - тип_ элемента[]; тип_ элемента[][];
    • задать значение
      • '{значение [, значение...]}'
      • ARRAY[значение [, значение..]]
      • ARRAY (подзапрос-с_одним_столбцом) или array_agg(выражение)
    • использование массив[индекс_элемента]
    • нумерация с единицы
    • создание среза - массив[нижний_индекс:верхний _индекс]
    • операции
      • сравнение, вхождение, пересечение, конкатенация
      • использование с ANY и ALL
    • циклы для перебора массива/slice

    Обработка ошибок#

    простая обработка вложенная обработка
    получение данных о exception детальная информация об exception
    • условие - либо имя либо код ошибки
    • условие OTHERS - перехватывает любую ошибку
    • необработанная ошибка отправляется клиенту
    • исключение вызывает rollback
    • передача исключения выше raise sqlstate 'имя_стейта'
    • Внутри секции EXCEPTION код ошибки можно получить из переменной SQLSTATE, а текст ошибки из переменной SQLERRM
    • Полный список информации об exception

    Триггеры#

    • События тригера для таблиц

      • INSERT, UPDATE, DELETE
        • before/after statement
        • before/after row
      • TRUNCATE
        • before/after statement

    • События тригера для представлений (views)

      • INSERT, UPDATE, DELETE - before/after statement - instead of row

    • BEFORE триггеры идеальны для проверки и изменения данных перед сохранением

    • AFTER триггеры полезны для действий, которые зависят от конечного состояния данных (например, логирование).

    • через WHEN можно установить дополнительный фильтр

    • триггеры отрабатывают в алфавитном порядке

    • триггерной функции можно передать параметры через TG_ARGV, сама функция не имеет параметров и возвращает trigger

    • Когда один триггер вызывает другой, можно случайно попасть в бесконечный цикл. Проверяйте такие ситуации с помощью TG_LEVEL

    • триггерная VOLATILE-функция видит результат работы триггеров на уровне BEFORE ROW или INSTEAD OF ROW.

    Raise#

    • можно настраивать отдельно выводы сервера
    • основные настройки логирования
      • log_min_duration_statement - время выполнения длинных команд (длинные команды логируются)
      • log_statement - какие команды включать
      • log_lock_waits - длинные ожидания
      • log_duration - время выполнения команд
      • log_temp_files - использование временных файлов
      • log_line_prefix - контекст сообщения
    • select * from pg_catalog.pg_settings where name like "1og%" - посмотреть настройки
    • select * from pg_stat_activity where backend type= 'client backend' and pid <> pe_backend_pid() - мониторинг процессов Postgresql
    • select pe_cancel_backend(); - остановить процесс по pid
